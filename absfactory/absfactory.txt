Abstract Factory
El patrón Abstract Factory proporciona una interfaz para crear familias de objetos relacionados o dependientes sin especificar sus clases concretas. Permite la creación de varios productos que están diseñados para trabajar juntos y delega la responsabilidad de creación a clases concretas que implementan la interfaz de fábrica abstracta.

Características principales:
- Crea familias de productos.
- Proporciona múltiples métodos de fábrica para crear diferentes productos.
- Usualmente se implementa con composición.
- Permite intercambiar familias de productos fácilmente.

Ejemplo:
Imagina que tienes diferentes tipos de botones y ventanas que deben funcionar juntos. El patrón Abstract Factory permite crear una familia de productos Button y Window que se usan conjuntamente.
